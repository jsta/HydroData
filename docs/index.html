Introduction

All environmental and hydrologic research begins with data; the question of what data is available; and the challenge of gathering it. HydroData is designed to help users find, get, visualize and use a range of Earth Systems data providing functions to automatically download; interact with; and share data.

Features

  • Access to 19 data sources with a consistent query
  • Helps find where data is available and get the data you want
  • Consistent data formats, projections, and documentation within all HydroData object
  • Interactive mapping and plotting via leaflet and dygraphs
  • Embed maps and data in R Markdown; Shiny; or export HTML files for easy documentation, reproducabilty, and sharing.
  • Export data for analysis in other platforms like python, ArcMap or QGIS

Installation

To install this package, run this command at your R prompt:

# to install the development version from Github, run
devtools::install_github("mikejohnson51/HydroData")

Once installed, you can use this package in the R console, within R Markdown documents, and within Shiny applications.

Basic Usage

You use HydroData with these basic steps:

  1. Define your Area of Interest (AOI).
  2. “Find” and “Get” data for that AOI using functions (e.g. findUSGS, getNED)
  3. Explore spatial and time series data interactively (e.g. explore, inspect)
  4. Repeat as desired.
  5. Analysis data or export for analysis in platform of choice

Here’s a basic example for all finding USGS stream flow gages in Santa Barbara County, California:

library(HydroData)
findUSGS(state = 'CA', county = "Santa Barbara") %>% explore()
## AOI defined as  boundary of Santa Barbara County, California
## 21 USGS gages found within AOI
## Returned list includes: USGS NWIS shapefile
